## 2. 실측 데이터

### 1688 (차단 확인)
```
https://www.1688.com/              HTTP 200, 93KB  (홈만 열림)
https://s.1688.com/.../search      HTTP 200, 2.5KB (captcha/punish 리다이렉트)
https://m.1688.com/                HTTP 200, 1.7KB (모바일도 차단)
```
응답 바디에 `x5secdata`, `_____tmd_____/punish` 토큰 확인 → 강한 봇 차단.

### 네이버 (모두 사용 가능, 무료)
```
openapi.naver.com/v1/datalab/search        → 401 (auth 필요, 엔드포인트 정상)
openapi.naver.com/v1/search/shop.json      → 401 (auth 필요, 엔드포인트 정상)
```

| API | 제공 데이터 | 한도 |
|-----|-------------|------|
| DataLab API | 검색 트렌드, 연령/성별 분포 | 25,000건/일 무료 |
| 쇼핑 검색 API | 키워드당 상품 목록/가격 | 25,000건/일, 쿼리당 최대 1,000건 |
| **검색광고 API (keywordstool)** | **월 검색량, 경쟁정도, 연관키워드** | 무료 |

## 3. 피벗된 아키텍처

```
┌──────────────────────────────────────────┐
│  ① 시스템: 한국 시장에서 기회 키워드 발굴  │
│     (네이버 3종 API 자동 수집)            │
│     → "이번주 주목 키워드 TOP 20"         │
└────────────────┬─────────────────────────┘
                 │
        [사용자에게 키워드 리스트 제시]
                 │
┌────────────────▼─────────────────────────┐
│  ② 사용자: 관심 키워드 선택 → 1688에서    │
│     해당 키워드 상품 1개 찾아 URL/가격 입력│
└────────────────┬─────────────────────────┘
                 │
┌────────────────▼─────────────────────────┐
│  ③ 시스템: 해당 상품의 최종 수익성 계산   │
│     + 네이버 경쟁 데이터로 종합 점수      │
│     + GO / 조건부 / 패스 판정             │
└──────────────────────────────────────────┘
```

### 시스템이 자동으로 하는 일 (①③)
- 네이버 검색광고 API로 **월 검색량 높은데 경쟁도 낮은** 키워드 발굴
- 네이버 DataLab으로 **최근 3개월 성장 추세** 양호한 키워드 필터
- 네이버 쇼핑으로 **상위 10개 평균 리뷰/가격대** 수집 → 경쟁 강도 산출
- 사용자 입력 상품에 대해: 환율·관세·수수료·광고비 계산 → 마진율/ROI/손익분기 산출

### 사용자가 하는 일 (②)
- 추천 키워드 중 관심 가는 것 선택
- 1688에서 해당 키워드 검색해 상품 1개 선정
- 입력: **URL, CNY 단가, MOQ** (3개 필드만)

## 5. 데이터 입력 최소화 — "3개 필드"

사용자 부담을 극한으로 줄이기 위해 입력은 **URL + CNY 단가 + MOQ**만.

| 자동 추론 | 방법 |
|-----------|------|
| KRW 환산가 | 실시간 환율 API |
| 관세·부가세 | 카테고리 기반 기본 테이블 |
| 국내 배송비 | 기본값 3,000원 (설정 가능) |
| 플랫폼 수수료 | 스마트스토어 5.5% 기본 |
| 예상 광고비 | 판매가 × 10% |
| 예상 판매가 | 네이버 쇼핑 상위 상품 평균 |
