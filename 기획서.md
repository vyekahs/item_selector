# 중국 소싱 기회 발굴 도구 기획서 (v2)

## 1. 프로젝트 개요

### 1.1 문제
중국에서 상품을 사입해 국내에서 팔려는 초심자는 "무엇을 팔지" 자체를 모른다. 감(感)과 유튜브 후기에 의존한 소싱은 재고로 쌓인다.

### 1.2 해결 방식 (하이브리드)
```
[시스템 자동]                              [사용자 수동]
한국 시장 데이터 다층 수집     →    "기회 키워드 TOP 20" 제시
· 수요 / 트렌드 / 실수입량 / 경쟁               ↓
                                      관심 키워드 선택
                                              ↓
                              1688에서 상품 찾아 URL·단가·MOQ 입력
                                              ↓
[시스템 자동]
스마트스토어 & 쿠팡 양쪽 수익성 계산 → 종합 점수 → GO/조건부/패스
```

**핵심**: 1688 자동 수집은 차단 확인됨 (쿠팡은 공식 API 사용 가능). **수요측 데이터로 "기회 탐지"만 자동화**, 공급측 상품은 사용자가 URL로 공급.

### 1.3 기존 툴과의 차별화

| 툴 | 초점 | 한계 |
|----|------|------|
| 블랙키위 | 네이버 검색량 | 소싱 수익 계산 없음 |
| 판다랭크 | 블로그/인스타/유튜브 언급 | 국내 마케팅용, 소싱 무관 |
| 아이템스카우트 | 스마트스토어 판매 분석 | 이미 팔리는 상품 위주, 선행 기회 약함 |
| **본 도구** | **관세청 수입량 + 네이버 + 트렌드 → 중국 소싱 적합도, 2채널 수익 비교** | — |

## 3. 데이터 소스 (모두 무료)

### 3.1 Tier 1 — 수요 (한국인이 뭘 찾는가)

| 소스 | 데이터 | 한도 |
|------|--------|------|
| 네이버 검색광고 API `keywordstool` | **월 검색량, 경쟁정도, 연관키워드** | 무료 |
| 네이버 DataLab API | 기간별 트렌드, 연령/성별/지역 | 25K/일 |
| 네이버 쇼핑 검색 API | 키워드당 상품 수, 가격대 | 25K/일 |
| 네이버 쇼핑인사이트 | 카테고리별 인기 검색어 TOP 500 | DataLab 내 |
| 네이버 블로그/카페 검색 API | 포스팅 수 = 관심 열기 | 25K/일 |

### 3.2 Tier 2 — 실체 (진짜 수입되고 있는가) 🔥

| 소스 | 데이터 | 활용 |
|------|--------|------|
| **관세청 품목별 국가별 수출입실적 API** | HS코드별 **월간 중국 수입량** 추이 | "실제 수입이 늘고 있는 품목" 필터 |
| 관세청 HS부호 DB | HS코드 ↔ 품목명 | 키워드-HS 매칭 |
| 한국수출입은행 환율 API | CNY/KRW 실시간 | 단가 환산 |

> 💡 **결정적 차별점**: 검색량만 늘면 거품일 수 있지만, **통관 실적도 함께 증가**하면 실수요.

### 3.3 Tier 3 — 선행지표 (곧 뜰 것인가)

| 소스 | 데이터 | 활용 |
|------|--------|------|
| YouTube Data API v3 | 키워드 영상 수, 조회수, 업로드 증가율 | "요즘 리뷰어들이 다루는 품목" |
| Google Trends (pytrends) | 글로벌 검색 트렌드 | 한국에 오기 전 조기 감지 |
| 네이버 블로그 최근 30일 포스팅 증가율 | 초기 바이럴 | 검색량 급등 전조 |

### 3.4 쿠팡 데이터 — ✅ 공식 API 확보

| API | 용도 | 제공 데이터 | 제약 |
|-----|------|------------|------|
| **쿠팡 파트너스 API** | **시장·경쟁 조사** | 카테고리 베스트셀러, 키워드 상품 검색, 가격, 평점, 리뷰 수, 로켓배송 여부 | **Search API 1시간 10회** |

→ 스마트스토어(네이버 쇼핑 API)와 **동등한 수준의 경쟁 데이터**를 쿠팡 측도 확보 가능.
→ Wing API(셀러용)는 본 도구 범위 외.

**1시간 10회 제약 대응**
- 키워드 조회 결과는 **24시간 PostgreSQL 캐시** (Redis 아님 — 영속성 필요)
- 기회 키워드 발굴 파이프라인에서 쿠팡 조회는 **Top 100 키워드로 한정** → 10시간에 걸쳐 야간 배치
- 사용자 실시간 상품 입력 시: 우선 캐시 조회 → 없으면 비동기 큐에 등록, 완료 시 알림

### 3.5 데이터 통합 구조
```
┌───────────────┐  ┌───────────────┐  ┌───────────────┐
│ [수요]        │  │ [실체]         │  │ [선행]         │
│ 네이버 키워드  │+ │ 관세청 수입량  │+ │ 유튜브/블로그  │
│ 네이버 쇼핑    │  │ HS코드 추이    │  │ 구글 트렌드    │
└───────┬───────┘  └───────┬───────┘  └───────┬───────┘
        └──────────────────┼──────────────────┘
                           ▼
                  [기회 점수 산출]
```

## 4. 기회 키워드 발굴 알고리즘

### 4.1 파이프라인
```
1) Seed 키워드 (카테고리별 200개 수동)
   ↓
2) 네이버 검색광고 API로 연관키워드 확장 (≈2,000개)
   ↓
3) 키워드별 지표 수집 (네이버 + 관세청 + 유튜브/블로그)
   ↓
4) HS코드 매핑 → 수입 통계 결합
   ↓
5) 기회 점수 → Top 20 추천
```

### 4.2 기회 점수 (100점)

| 영역 | 배점 | 지표 |
|------|------|------|
| 수요 크기 | 25 | 월 검색량 (로그 정규화) |
| 수요 성장 | 20 | 최근 3개월 검색량 증가율 |
| 경쟁 공백 | 20 | (검색량 ÷ 경쟁 상품 수) 블루오션 지수 |
| **수입 실체** | 20 | 관세청 월 수입량 증가율 |
| 트렌드 선행 | 10 | 유튜브/블로그 언급 증가율 |
| 안정성 | 5 | 계절성, 규제 감점 |

### 4.3 자동 제외 필터
- ❌ KC/전안법/식약처 인증 필수
- ❌ 계절성 지수 > 2.5
- ❌ 경쟁 상품 평균 리뷰 > 1,000 (레드오션)
- ❌ 관세청 수입량 3개월 연속 감소

## 5. 상품 점수 엔진 (사용자 입력 후)

### 5.1 사용자 입력 (3 필드)
- 1688 상품 URL
- CNY 단가 (또는 범위)
- MOQ

### 5.2 **스마트스토어 & 쿠팡 양쪽 계산**

**공통 원가**
```
총 원가     = 사입가×MOQ + 국제배송 + 관세 + 부가세
개당 원가   = 총 원가 ÷ MOQ
```

**채널별 수익 (병행 표기)**

| 항목 | 스마트스토어 | 쿠팡 |
|------|-------------|------|
| 플랫폼 수수료 | 5.5% | 카테고리별 4~10.8% (테이블 내장) |
| 결제 수수료 | 3.74% | 포함 |
| 예상 광고비 | 판매가 × 10% | 판매가 × 15% (경쟁 치열) |
| 기본 판매가 | 네이버 쇼핑 상위 평균 | 쿠팡 상위 평균 (사용자 확인) |
| 배송 모델 | 일반 | 일반 또는 로켓그로스 선택 |

**산출 결과**
```
┌────────────┬──────────────┬──────────────┐
│            │ 스마트스토어   │ 쿠팡         │
├────────────┼──────────────┼──────────────┤
│ 개당 순이익 │ 24,400원     │ 19,100원     │
│ 마진율     │ 64%          │ 50%          │
│ ROI        │ 128%         │ 98%          │
│ 손익분기   │ 17개         │ 21개         │
└────────────┴──────────────┴──────────────┘
→ 추천 채널: 스마트스토어 (순이익 27% 높음)
→ 병행 전략: 쿠팡도 흑자, 동시 판매 권장
```

### 5.3 종합 점수 (100점)

| 영역 | 배점 |
|------|------|
| 기회 점수 (키워드) | 40 |
| **수익성 (양 채널 평균 ROI·마진)** | 35 |
| 리스크 (MOQ 부담, 손익분기) | 15 |
| 공급 안정성 (사용자 체크) | 10 |

**판정**: 80+ GO ✅ / 60~79 조건부 ⚠️ / <60 패스 ❌

```
│ 🎯 이번주 중국 소싱 기회 TOP 20                │

│   스마트스토어 평균가 38K / 쿠팡 42K           │
│   [1688에서 찾기 →] [상품 입력]                │
```

### 6.2 상품 상세 — 2채널 판정
```
키워드: 고양이 자동급수기

──────────────────────────────
[레이더차트: 기회 85 / 수익 72 / 리스크 68 / 안정 80]

수익 시뮬레이션                스마트스토어   쿠팡
  개당 원가                    8,200원       8,200원
  평균 판매가                  38,000원      42,000원
  개당 순이익                  24,400원      19,100원
  마진율                       64%           50%
  ROI                          128%          98%
  손익분기 (45개/월 가정)      17개          21개

🏆 추천: 스마트스토어 (우선)
   → 병행: 쿠팡도 GO (순이익 격차 27%)

리스크
  🚩 MOQ×단가 41만원 (예산 내 ✅)
  ⚠️ 쿠팡 상위 경쟁사 리뷰 평균 340개
```

### 6.3 대시보드 — 입력 이력
- 입력 상품들을 표로 비교 (채널별 점수/마진/ROI 나란히)
- 60일 후 "실제 구매 여부 / 실제 매출" 피드백 입력

## 7. 개발 단계

### Phase 1 — 키워드 발굴 MVP (2주)
- 네이버 3종 API 연동
- 반려동물 카테고리 Seed 200개 → Top 20 키워드
- 로컬 웹페이지 노출

### Phase 2 — 실체 데이터 결합 (2주)
- 관세청 수출입 API 연동
- HS코드 ↔ 키워드 매핑 테이블
- 수입량 지표 점수 반영

### Phase 3 — 상품 입력 & 2채널 판정 (3주)
- URL + 단가 + MOQ 입력 UI
- 환율/관세/수수료 자동 계산
- **스마트스토어 & 쿠팡 API로 양쪽 실시간 경쟁 데이터**
- 2채널 병행 수익 비교 표시

### Phase 4 — 선행지표 & 학습 (4주)
- 유튜브/블로그/구글 트렌드 결합
- 피드백 루프 (실제 매출 입력) → 가중치 자동 조정

## 8. 기술 스택

| 영역 | 선택 |
|------|------|
| 언어 | Python (수집·API), TypeScript (프론트) |
| 프레임워크 | FastAPI + Next.js |
| 스케줄러 | APScheduler (FastAPI 내) 또는 별도 cron 컨테이너 |
| **DB** | **PostgreSQL** (처음부터) |
| 캐시 | Redis |
| **배포** | **홈서버 + Docker** (docker-compose로 다중 서비스 구성) |

### 8.1 컨테이너 구성 (docker-compose)
```
services:
  web        → Next.js (프론트)
  api        → FastAPI (백엔드)
  collector  → 데이터 수집 워커 (네이버/관세청/쿠팡/유튜브)
  scheduler  → 배치 스케줄러
  postgres   → 데이터 저장
  redis      → 캐시 / 작업 큐
  nginx      → 리버스 프록시 + HTTPS
```

### 8.2 홈서버 운영 고려
- 포트 포워딩 또는 Cloudflare Tunnel로 외부 접근
- 데이터 볼륨 마운트 (PostgreSQL 영속성)
- 자동 재시작 정책 (`restart: unless-stopped`)
- 로그 관리 (용량 제한 설정)
- 백업: PostgreSQL dump 주기적 스냅샷
